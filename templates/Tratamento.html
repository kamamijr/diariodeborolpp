Tópico:Tratamento de Execeções em C#

O tratamento de exceção é uma abordagem na programação que permite lidar com situações anormais ou erros que podem ocorrer durante a execução de um programa. Ele ajuda a prevenir falhas catastróficas e oferece a oportunidade de recuperar e lidar com erros de maneira controlada.

Subtópico: Projeto de Exceção

O projeto de exceção envolve a identificação e o planejamento de como lidar com possíveis erros em um programa. Isso inclui a definição de exceções personalizadas para cenários específicos, permitindo um tratamento mais adequado e compreensível de erros.

Subtópico: Tratamento de Exceção

O tratamento de exceção ocorre quando um bloco de código é envolto em um bloco "try-catch". O bloco "try" contém o código que pode gerar exceções, enquanto o bloco "catch" captura e trata essas exceções, evitando que o programa seja encerrado abruptamente.
Claro, vou fornecer um pouco mais de informações sobre o tratamento de exceções em C#.
Bloco try: Este bloco envolve o código no qual você suspeita que exceções podem ocorrer. Ele é usado para monitorar exceções durante a execução do código contido nele.

try
{
// Código que pode gerar exceções
}
Bloco catch: Este bloco é usado para capturar exceções que ocorreram no bloco try. Você pode ter vários blocos catch para tratar diferentes tipos de exceções.

catch (TipoDeExcecao1 ex)
{ // Trate a exceção TipoDeExcecao1
} catch (TipoDeExcecao2 ex)
{ // Trate a exceção TipoDeExcecao2
} Bloco finally: O bloco finally é opcional e contém código que será executado, independentemente de ocorrer ou não uma exceção no bloco try. É frequentemente usado para liberar recursos ou realizar operações de limpeza.

finally
{
// Código a ser executado sempre
}
Exceções Personalizadas: Além das exceções incorporadas no C#, você pode criar suas próprias exceções personalizadas para cenários específicos. Isso ajuda a tornar o tratamento de exceção mais significativo e legível.

public class MinhaExcecaoPersonalizada : Exception
{
public MinhaExcecaoPersonalizada(string mensagem) : base(mensagem)
{
// Construtor da exceção personalizada
}
}
Lançamento de Exceções: Você pode lançar exceções manualmente usando a palavra-chave throw. Isso é útil quando deseja sinalizar um erro específico.

if (algumaCondicao)
{ throw new MinhaExcecaoPersonalizada("Ocorreu um erro específico.");
}
Hierarquia de Exceções: Em C#, as exceções são organizadas em uma hierarquia de classes de exceção. A classe Exception é a classe base para todas as exceções. Isso permite que você capture exceções específicas antes de capturar exceções mais genéricas. try
{
// Código que pode gerar exceções
}
catch (MinhaExcecaoPersonalizada ex)
{
// Trate a exceção personalizada
}
catch (Exception ex)
{
// Trate outras exceções não específicas
}
O tratamento de exceções em C# é uma prática essencial para garantir que seu código seja resiliente a erros e ofereça uma experiência mais amigável ao usuário ao lidar com situações inesperadas. Além disso, o uso de exceções personalizadas ajuda a tornar o código mais claro e legível, facilitando a manutenção e o diagnóstico de problemas.

EXEMPLOS DE CODIGO

Exemplo de Código: Projeto de Exceção em C#

class SaldoInsuficienteException : Exception {
public SaldoInsuficienteException() { }
public SaldoInsuficienteException(string message) : base(message) { }
}

class ContaBancaria {
private double saldo;

public ContaBancaria(double saldoInicial) {
if (saldoInicial < 0)
throw new SaldoInsuficienteException("Saldo inicial inválido");
saldo = saldoInicial;
}

public void Sacar(double valor) {
if (valor > saldo)
throw new SaldoInsuficienteException("Saldo insuficiente");
saldo -= valor;
}

Exemplo de Código: Tratamento de Exceção em C#

class Program {
static void Main(string[] args) {
try {
int numerador = 10;
int denominador = 0;
int resultado = numerador / denominador;
Console.WriteLine(resultado);
} catch (DivideByZeroException ex) {
Console.WriteLine("Erro: Divisão por zero");
} catch (Exception ex) {
Console.WriteLine("Erro: " + ex.Message);
}
}
}
